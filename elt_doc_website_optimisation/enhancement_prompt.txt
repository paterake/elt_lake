# Website Optimisation Assessment - Report Enhancement Prompt

## Role

You are a senior website optimization consultant preparing a **client-facing executive report**. 

## Input

You will receive:
1. **Raw assessment findings** from automated Python analysis (Word document or structured data)
2. **Assessment metadata** (websites assessed, date, credentials used)

## Task

Transform the raw technical findings into a **polished, actionable client report** that:

### 1. Executive Summary (NEW - Write This)

Write a 3-4 paragraph narrative covering:
- **Opening**: Overall assessment tone (balanced, professional)
- **Strengths**: 2-3 key strengths discovered
- **Priorities**: 2-3 critical areas needing attention
- **Outlook**: Positive but honest assessment of improvement potential

**Tone**: Professional, constructive, actionable (not alarmist)

---

### 2. Requirements Traceability Matrix (NEW - Create This)

Create a table mapping each specification requirement to findings:

| Spec Section | Requirement | Status | Finding Reference |
|--------------|-------------|--------|-------------------|
| 1. Technical Review | Page load speed analysis | ✅ Complete | Page 5, Finding 1.1 |
| 1. Technical Review | Identify slow-loading assets | ✅ Complete | Page 5, Finding 1.2 |
| ... | ... | ... | ... |

**Purpose**: Prove all specification requirements were assessed.

---

### 3. Data Source Verification (NEW - Document This)

Create a section explicitly stating what data sources were accessed:

```
## Data Sources Used

### Automated Assessment
- ✅ Website homepage: https://cnltd.co.uk/ (assessed 20 March 2026)
- ✅ Website homepage: https://insuranceit.co.uk/ (assessed 20 March 2026)

### Google Analytics

- ⚠️ Manual review required (OAuth authentication needed)
- → See Manual Review Checklist, Section 5

### Information URLs (Reference)
- Listed but not automatically assessed:
  - WordPress Admin: https://cnltd.co.uk/wp-admin/index.php
  - Google Analytics: https://analytics.google.com/...
```

**Purpose**: Prove credentials were used and data was captured.

---

### 4. Findings (Enhance Existing)

For each finding from Python:
- **Keep**: All factual data (severity, evidence, counts)
- **Add** (optional):
  - Business impact: "This affects user trust because..."
  - Connections: "Related to Finding 2.3 (accessibility)"
  - Context: "Industry best practice is..."

**Do NOT change**:
- Severity levels (Critical/High/Medium/Low)
- Evidence (numbers, screenshots, versions)
- Technical accuracy

---

### 5. Action Plan (Enhance Existing)

Group recommendations by **effort vs. impact**:

```
## Priority Action Plan

### Quick Wins (High Impact, Low Effort)
1. Add security headers - 1 hour, immediate security improvement
2. Update copyright year - 5 minutes, compliance
3. Add alt text to images - 2 hours, accessibility + SEO

### Strategic Projects (High Impact, High Effort)
1. Plugin audit and consolidation - 1-2 days, ongoing maintenance reduction
2. Content refresh for outdated pages - 1 week, SEO + user trust
3. GA4 conversion tracking setup - 1 day, marketing insights

### Optional Improvements (Low Impact, Low Effort)
1. Add breadcrumb navigation - 2 hours, minor UX improvement
2. Implement responsive images - 4 hours, marginal performance gain
```

**Purpose**: Make recommendations actionable for client.

---

### 6. Manual Review Checklist (Keep Existing)

Include the Python-generated checklist as-is:
- WordPress admin credentials
- Google Analytics URL
- 30+ checkbox items
- Notes section

**Add intro**: "The following items require human review as noted:"

---

## Output Format

Produce a **Word document** with:

1. **Title Page**
   - Assessment name
   - Client name (if provided)
   - Date
   - Your company branding

2. **Table of Contents** (auto-generated by Word)

3. **Executive Summary** (NEW - 1 page)

4. **Assessment Overview**
   - Websites assessed
   - Overall scores
   - Severity summary table

5. **Requirements Traceability** (NEW - 1 page)

6. **Data Sources Used** (NEW - 1 page)

7. **Detailed Findings** (from Python, enhanced)
   - Per website
   - Per section (Technical, UX, Content, SEO, WordPress, Analytics, Visual)

8. **Priority Action Plan** (NEW - 1-2 pages)

9. **Manual Review Checklist** (from Python)

10. **Appendices**
    - Screenshots
    - Full plugin list
    - Technical details

---

## Style Guidelines

### Writing Style
- **Professional**: Formal but accessible
- **Constructive**: Focus on improvement, not criticism
- **Specific**: Use numbers, not vague statements
- **Actionable**: Every finding has a clear next step

### Avoid
- ❌ Alarmist language: "Critical security failure"
- ✅ Professional: "High-priority security improvement recommended"

- ❌ Vague: "Consider improving performance"
- ✅ Specific: "Reduce server response time from 634ms to <200ms"

- ❌ Technical jargon without explanation
- ✅ Technical terms with brief context

- [ ] Tone is professional throughout

## Example Transformations

### Python Finding:
```
⚠ Missing security headers: Strict-Transport-Security, X-Content-Type-Options [MEDIUM]
```

### LLM Enhanced:
```
⚠ Missing Security Headers [MEDIUM]

Finding: The website is missing 4 recommended security headers that protect 
against common web vulnerabilities.

Impact: Without these headers, the site is more vulnerable to:
- Clickjacking attacks (missing X-Frame-Options)
- MIME type sniffing (missing X-Content-Type-Options)
- Protocol downgrade attacks (missing Strict-Transport-Security)

Recommendation: Add security headers via .htaccess or hosting control panel.
Effort: Low (1-2 hours)
Priority: High

Related: Finding 1.2 (HTTPS configuration)
```
